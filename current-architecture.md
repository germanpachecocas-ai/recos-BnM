# Current Architecture — recos-BnM

## Project Overview

Platforma de recomendación tipo Tinder para contenido multimedia. Los usuarios hacen swipe (like/dislike) a películas, series y libros, y el sistema genera un feed personalizado basado en afinidad de géneros y scoring normalizado.

## Tech Stack

| Layer | Technology |
|---|---|
| **Ingest (data pipeline)** | Python 3.12+, requests, google-cloud-firestore |
| **API / Scoring** | Node.js, Google Cloud Firestore |
| **Database** | Firestore (emulator local, Cloud Firestore prod) |
| **Auth** | Firebase Authentication (planned) |
| **Infrastructure** | Cloud Run (ingest job), Cloud Scheduler (cron) |
| **Local emulator** | Firebase Emulator Suite, Java 21 (Temurin) |

## Firestore Collections (SCHEMA.md)

| Collection | Document ID | Purpose | Auth |
|---|---|---|---|
| `users/{userId}` | Firebase Auth UID | Profile, preferences, avatar | read/write: owner only |
| `content/{contentId}` | `source_externalId` | Catalog (movies, series, books), read-only | read: authed, write: none |
| `swipes/{swipeId}` | Auto-generated | Like/dislike log | read/write: owner only |
| `collections/{collectionId}` | Auto-generated | Custom lists (favorites, watch later) | read/write: owner only |

### content/ Document Fields

| Field | Type | Description |
|---|---|---|
| `title` | string | Content title |
| `type` | string | `"movie"` / `"series"` / `"book"` |
| `creator` | array of strings | Directors (movies) or authors (books) |
| `genres` | array of strings | Genre tags |
| `description` | string | Synopsis |
| `posterUrl` | string | Poster or cover image URL |
| `year` | number | Release or publication year |
| `rating` | number | Average rating (0–10 scale) |
| `popularity` | number | Popularity metric (0 for books) |
| `source` | string | `"TMDB"` / `"Google Books"` |
| `whereToWatch` | array of strings | Streaming platforms (movies) |

## Scoring Formula (`api/src/services/scoring.js`)

```
score = 0.7 × norm(popularity) + 0.3 × norm(rating)
```

- **Normalisation:** min-max over candidate set at query time
- **Edge case:** if max === min → score = 0.5
- **Genre Affinity (optional):** multiplier > 1 boosts preferred genres, < 1 penalises disliked
- **Books:** popularity = 0.0 always (Google Books doesn't provide it), rating 0–5 × 2 → 0–10 scale

- TMDB attribution required in UI: *"This product uses the TMDB API but is not endorsed or certified by TMDB."*
- Emulator requires Java 21 at specific path
